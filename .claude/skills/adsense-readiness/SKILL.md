# 애드센스 심사 준비 (Readiness) 체크리스트

## 왜 이 스킬이 필요한가 (Why-first)

애드센스 거절은 대부분 **콘텐츠 품질·필수 페이지 누락·정책 위반**이라는 예측 가능한 사유에서 나온다. 개발 착수 전에 이 기준을 **요구사항으로 못박아** 두면, 나중에 재작업 없이 심사 통과 확률 80% 이상을 겨냥할 수 있다. 이 스킬은 PRD에 그 기준을 기입하고, 검증 단계에서 정책 최신성을 확인하는 데 쓴다.

## 승인 기준 (요구사항으로 기입할 것)

- **오리지널·유용한 콘텐츠**: 사용자에게 실질 가치를 주는 고유 콘텐츠. 타 사이트 복제·스크래핑 재게시 금지.
- **충분한 분량·페이지 수**: 심사에 필요한 핵심 콘텐츠 페이지 최소치를 확보(주제별 깊이 있는 문서 다수). 빈 껍데기 사이트 금지.
- **명확한 네비게이션·구조**: 메뉴·카테고리·내부 링크로 탐색 가능한 사이트 구조.
- **필수 정책·정보 페이지**: **개인정보처리방침(Privacy Policy)**, **이용약관(Terms)**, **문의(Contact)**, **소개(About)**.
- **정상 작동·완성도**: 깨진 링크·미완성 페이지·플레이스홀더 없음. 모바일 반응형·빠른 로딩.
- **도메인·연결성**: 사이트가 공개 접근 가능하고 크롤링이 허용됨(robots·noindex 오설정 금지).

## 흔한 거절 사유 (회피 대상)

- **저품질/자동생성 콘텐츠**: 사람이 검수하지 않은 대량 자동생성·AI 스팸·얇은 콘텐츠. → 수집 데이터라도 편집·정제·부가가치를 더한다.
- **콘텐츠 부족(Under construction)**: 페이지 수·분량 미달.
- **복제·저작권 침해**: 타 출처 콘텐츠 무단 복제, 이미지 저작권 위반.
- **정책 위반 콘텐츠**: 성인물·폭력·불법·유해·기만적 콘텐츠.
- **필수 페이지 부재**: 개인정보처리방침·문의 등 누락.
- **광고 배치 정책 위반**: 콘텐츠보다 광고가 많거나, 오클릭 유도 배치.

## 콘텐츠 품질 기준

- 자동 수집(cron API·Playwright 크롤링) 데이터를 쓰더라도 **원본 그대로 게시하지 말고** 구조화·요약·큐레이션·해설 등 **편집적 부가가치**를 더해 오리지널리티를 확보한다.
- E-E-A-T(경험·전문성·권위·신뢰) 관점에서 출처 표기·작성 기준을 갖춘다.
- 각 페이지가 단독으로 사용자 질문에 답할 수 있을 만큼 충실할 것.

## PRD/검증에서의 사용법

- **prd-author**: 위 승인 기준·필수 페이지·품질 기준을 PRD의 "애드센스 준비 요구사항" 섹션에 요구사항으로 명문화한다. 자동 수집 콘텐츠에는 반드시 편집 부가가치 요구를 붙인다.
- **tech-verifier**: 애드센스 정책은 변경될 수 있으므로, 참조한 정책 기준이 **현행인지** 공식 문서/WebSearch로 확인하고 죽은 링크·구정책 인용을 걸러낸다.

## 자체 통과 확률 점검(간이)

필수 페이지 4종 · 오리지널 콘텐츠 · 분량 충족 · 네비게이션 · 정책 위반 없음 — 이 5개가 모두 요구사항에 반영되고 검증 가능한 완료 기준이 붙었는지 확인한다. 하나라도 빠지면 통과 확률이 떨어지므로 PRD를 완료로 보지 않는다.
