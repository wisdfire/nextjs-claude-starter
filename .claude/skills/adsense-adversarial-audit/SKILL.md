# 애드센스 적대적 검증 (Adversarial Audit)

> **요건의 SSOT는 `adsense-readiness` 스킬(MUST 10항)이다.** 이 스킬은 **요건을 정의하지 않고, 요건이 실제로 충족됐는지를 적대적으로 검증하는 절차**를 정의한다. 두 문서의 내용이 어긋나면 `adsense-readiness`가 이긴다.

## 왜 필요한가 — 기계 검증의 사각지대

`qa-inspector`의 Grep 게이트는 **존재/부재**만 판정한다. 그런데 애드센스 거절의 큰 축은 **정성 판정**이다.

| 기계 검증이 내는 결론 | 심사관이 내리는 결론 |
|---|---|
| `app/privacy/page.tsx` 존재 → 통과 | 내용이 템플릿 껍데기, 구글 지정 문구 없음 → **거절** |
| 광고 슬롯이 저가치 라우트에 없음 → 통과 | 콘텐츠 페이지 자체가 수집 데이터 나열 → **거절(복제·저가치)** |
| 링크 404 = 0 → 통과 | 링크는 살아있으나 도착지가 빈 목록 → **거절(No content)** |
| "준비 중" 문자열 없음 → 통과 | 문구만 없을 뿐 실질은 미완성 화면 → **거절** |

**존재는 충족이 아니다.** 그 간극을 메우는 것이 이 단계다.

## 실행 조건

- **시점**: Phase 4.3 검증 게이트(lint·build·test·기계 QA) **통과 이후**, 배포·완료 **이전**. 깨진 빌드를 심사해봐야 의미가 없고, 배포 후에 발견하면 늦다.
- **전제**: `npm run dev`(또는 프리뷰 배포)가 떠 있어 **실제 화면을 렌더할 수 있어야** 한다. 코드 리뷰만으로는 이 단계를 수행할 수 없다 — 렌더 불가면 그 사실을 보고하고 **통과 처리하지 않는다**.
- **면제**: 사용자가 애드센스를 명시적으로 면제한 경우에만 건너뛰고, 면제 사실을 보고에 남긴다.

## 절차

### Step 0 — 정책 현행성 재확인 (매 감사 1회, 필수)

`adsense-readiness`는 기준일이 박힌 스냅샷이다. **낡은 기준으로 내린 판정은 무효다.**

1. 스킬 문서의 기준일과 오늘 날짜의 간격을 확인한다.
2. 각 렌즈 담당은 자기 렌즈에 걸린 MUST의 **출처 URL을 WebFetch로 실제로 연다**. ①살아있는지 ②스킬 요약이 현재 문서와 일치하는지 대조한다.
3. WebSearch로 최근 정책 변경(Publisher Policies·검색 스팸 정책·CMP 적용 범위)을 확인한다.
4. **불일치를 발견하면** 감사 리포트 최상단에 `[정책 갱신 필요]`로 보고하고, `adsense-readiness` 갱신을 리더에게 요청한다. 감사는 **현행 정책 기준으로** 계속한다.

> 확인하지 못한 것은 "확인 불가"로 남긴다. **문서를 못 연 것을 "변경 없음"으로 적지 않는다.**

**현재 기준선(2026-07-20 전수 검증 완료)** — 다음은 이미 확인됐으니 재확인 우선순위를 낮춰도 된다:

| 항목 | 확인 결과 |
|---|---|
| 출처 12종 생존 | 죽은 링크 **0건** |
| 2026년 정책 변경 | **없음** — [AdSense 체인지로그](https://support.google.com/adsense/answer/9336650)·[Publisher Standards 체인지로그](https://support.google.com/publisherpolicies/answer/10852414) 직접 확인. 2026년 변경은 광고 기술 파트너·Multiplex 계수 등 **운영 영역** |
| ads.txt 의무화 | **없음** — "not mandatory, but highly recommended" 그대로 |
| CMP 지역 확대 | **없음** — EEA·UK·스위스 그대로 |
| 2026-03 core update | **검색 랭킹 업데이트**이지 애드센스 심사 정책 변경이 아니다 |

**커뮤니티 신호는 요건이 아니다.** "low value content는 신규 사이트 대상 포괄 거절 사유다", "약한 페이지를 삭제하면 승인율이 오른다", "'site not ready'는 완성돼 보이지 않는다는 뜻이다" 같은 서술은 전부 3자 블로그이며 공식 확인 불가다. **렌즈 우선순위 힌트로만 쓰고 지적의 근거로 인용하지 않는다.** 특히 **"300~400자 미만은 thin"은 무근거이며 채택 금지**다.

### Step 1 — 렌즈 배정 (독립·병렬)

렌즈를 **서로 다른 에이전트에 배정해 독립적으로** 심사시킨다. 한 에이전트가 전부 보면 앞에서 내린 "통과" 판정에 이후 판정이 끌려간다(앵커링). 각 렌즈는 다른 렌즈의 결론을 모른 채 시작한다.

| # | 렌즈 | 심사 질문 | 연관 MUST |
|---|---|---|---|
| **L1** | **콘텐츠 가치** | 이 페이지는 **사용자에게 독자적 가치**를 주는가? 다른 곳에서 얻을 수 없는 정보인가? | 5·6·7·10 |
| **L2** | **정책·법적 준수** | 개인정보처리방침이 **구글 지정 문구를 실제로 담고** 있는가? 광고 배치·CMP·HTTPS가 정책에 맞는가? | 1·2·3·8 |
| **L3** | **크롤러·완성도** | 구글이 이 사이트를 **크롤링·평가할 수 있는가**? 사용자가 약속된 정보에 **실제로 도달**하는가? | 3·4·5·9 |

각 렌즈는 **`reject` / `approve-with-risk` / `approve`** 중 하나를 낸다.

### Step 2 — L1 콘텐츠 가치 렌즈 (★ 이 프로젝트의 최대 거절 리스크)

수집 데이터를 렌더하는 제품이므로 **복제·저가치 콘텐츠가 1순위 거절 사유**다. 아래를 **페이지 단위로** 심사한다.

**심사 질문 (각각 증거와 함께 답한다)**

1. **출처 삭제 테스트 (가장 날카로운 질문)** — **"이 페이지에서 원본 출처의 내용을 지우면 사용자가 잃는 것이 무엇인가?"** 아무것도 없다면 그 페이지는 replicated content다. "정렬·필터가 다르다"는 편집 부가가치가 아니다. 요약·비교·집계·해설·큐레이션 중 **무엇을 더했는지 지목할 수 없으면 `blocker`**. (MUST 7)
   - **검증**: 수집 원본 필드(`raw_*`·DB 원문 컬럼)가 가공 없이 그대로 렌더되는 경로를 Grep하고, 그 경로마다 집계·비교·요약·구조화 중 **최소 1개의 파생 산출물이 DOM에 실재**하는지 대조한다.
2. **사람 검토 증명 테스트** — "사람이 검토했다"를 **증명할 수 있는가?** 데이터 모델에 검토 상태·게시 승인 단계(`reviewed_at`·`published_by` 등)가 없으면 **"대량 자동생성"과 구별할 방법이 없다**. 페이지 수가 데이터 행 수를 따라 자동으로 늘어나는데 승인 게이트가 없으면 `scaled content abuse` 리스크로 지적한다. (MUST 6)
3. **빈 상태 테스트** — 데이터가 없을 때 이 페이지는 무엇을 보여주는가? **빈 목록·스켈레톤만 남는 라우트는 "No content"** 다. 실제로 빈 데이터로 렌더해 확인한다. (MUST 5)
4. **미완성 테스트** — 문구가 아니라 **실질**로 판정한다. "준비 중" 텍스트가 없어도, 클릭할 수 없는 버튼·값이 비어 있는 카드·설명 없는 지표는 미완성 화면이다. 구글 거절 사유 원문: **"site was deemed to be 'under construction'"**. (MUST 5·10)
   - **검증**: 렌더 결과에서 `준비 중`·`Coming soon`·`Under construction`·`TODO`·`Lorem ipsum`·`placeholder` 문자열 1건이라도 → `blocker`.
   - **검증**: 미해결 번역 키가 화면에 노출되는지 — 텍스트 노드에서 `/^[A-Z]\w+\.\w+$/` 패턴 탐지.
5. **publisher-content 0 테스트** — 각 라우트의 렌더 결과에서 **publisher-content가 0인 페이지**를 목록화한다. ⚠️ **글자수 임계값을 만들지 마라** — 구글은 "too little text"의 수치 기준을 제시하지 않는다. **0인가 아닌가**로만 기계 판정한다. (MUST 10)
6. **기계번역 테스트** — 영어 화면의 **본문**이 사람 검토 없이 번역돼 있지 않은가? UI 문자열(`messages/*.json` = 코드가 정의한 것) 번역은 정상이지만, **DB·수집 유래 텍스트가 번역 파이프라인에 유입되면 `blocker`**다. (MUST 6 · `i18n-localization`)
   - **검증**: 본문이 미번역인 문서의 영어 라우트에 `hreflang="en"`이 붙지 않고 `canonical`이 한국어 URL을 가리키는지.
7. **템플릿 파생(formulaic) 테스트 (★ streamprice 학습 — 출처 삭제 통과가 뎁스 통과는 아니다)** — Q1(출처 삭제 테스트)을 통과해 파생 산출물이 **非0**인 것은 **최소 문턱일 뿐 뎁스 판정이 아니다.** 같은 라우트 패턴의 렌더 결과 **3개 이상을 나란히 놓고** 편집 레이어를 비교한다: **숫자·고유명사만 바뀐 동일 문장**이 반복되면(예: "N번째로 낮습니다"가 전 페이지 동형), 편집 레이어가 있어도 **대량 자동생성과 구별되지 않는다**. 데이터 조건(신규/인상/동률/결측)에 따라 해설 구조가 갈라지지 않으면 최소 `risk`, **전 사이트가 단일 템플릿 한 줄뿐이면 `blocker`**. (MUST 6 · `adsense-readiness` §편집 뎁스)
   - **검증**: 표본 3개 이상에서 편집 문단을 추출해 **토큰 diff**한다. 숫자·고유명사만 다르고 문장 골격이 동일하면 formulaic으로 판정하고, 지적에 **템플릿 문자열과 이를 공유하는 라우트 목록**을 증거로 첨부한다.
8. **편집 뎁스 유형 커버리지 테스트** — 사이트 전체에 `adsense-readiness` §편집 뎁스의 **4종 레이어(①페이지별 해설 심화 ②에버그린 가이드/글 ③큐레이션·상호연결 ④방법론·FAQ/용어)** 중 실질적으로 존재하는 것을 목록화한다. 광고를 붙이는 콘텐츠 화면이 **유형 1의 템플릿 한 줄에만 의존하고 나머지 3종이 전무**하면, 데이터가 얇은 조합에서 저가치로 무너질 구조적 리스크로 `risk` 이상 지적한다. ⚠️ **글자수·글 수 임계값을 만들지 마라** — 유형 커버리지와 비-템플릿성으로만 판정한다(무근거 수치 창작 금지 원칙과 동일).

**보완 지시의 형식** — L1의 지적은 "품질을 높여라"로 끝나면 아무 일도 일어나지 않는다. **무엇을 더할지**를 `adsense-readiness` §편집 뎁스 4종 레이어 중 무엇을 어느 화면에 추가할지로 지목한다:

> ❌ "콘텐츠 가치가 부족함"
> ✅ "`/jobs/[id]` — 수집 필드를 나열만 함. **동일 직무군 평균 대비 비교 1줄 + 마감까지 남은 기간 계산 + 유사 공고 3건 큐레이션**을 서버에서 생성해 추가할 것 (backend-engineer: 집계 쿼리, frontend-engineer: 렌더)"

### Step 3 — L2 정책·법적 렌즈

1. **개인정보처리방침 실질 검사** — 존재가 아니라 **내용**을 읽는다. 구글 지정 4요소(제3자 쿠키 광고·구글 광고 쿠키·`google.com/settings/ads` opt-out 안내·제3자 애드네트워크 목록)가 **문장으로 실재**하는가. 하나라도 없으면 `blocker`. (MUST 1)
2. **도달성** — 한국어·영어 **양쪽 화면**에서 푸터 링크로 실제로 열리는가. 영어 화면 404면 사실상 미제공이다. (MUST 1·9)
3. **CMP** — EEA·UK·스위스 트래픽을 받는다면 인증 CMP(IAB TCF) 계획이 문서·구현에 있는가. 미구현이면 최소 `risk`. (MUST 2)
4. **광고:publisher-content 비율** — **분모의 정의가 핵심이다.** `publisher-content`는 이미지·비디오·게임·본문 텍스트·발행자 관리 UGC이며, **여백·헤더/푸터·다른 콘텐츠로의 링크는 포함되지 않는다.**
   - **적대적 질문(이 프로젝트에서 가장 걸리기 쉬움)**: "카테고리 인덱스·태그 목록·검색 결과는 **링크 목록뿐이라 publisher-content가 0**인데 광고를 붙였는가?" → 붙였으면 `blocker`(MUST 8 위반이자 MUST 5의 저가치 화면).
   - **검증**: 광고 허용 라우트마다 `광고 슬롯 수` vs `publisher-content 블록 수`를 세되 **헤더·푸터·내비·링크 목록·여백을 분모에서 제외**하고 센다.
5. **광고 배치·라벨** — **렌더된 화면에서** 본다. (MUST 8)
   - 라벨이 `Advertisements`·`Sponsored Links` 외의 값이면 `blocker` (**"광고"·"스폰서" 같은 자체 문구도 금지**).
   - 광고 슬롯과 내비게이션/버튼의 DOM 인접성·렌더 좌표 간섭.
   - 광고 컨테이너에 대한 프로그램적 refresh 호출 0건.
   - **"resources"·"helpful links" 류 제목 바로 아래 광고 배치 없음**(오도하는 제목).
   - **클릭 유도 문구 없음** ("Feel free to click an ad" 류).
6. **HTTPS** — 프리뷰/프로덕션 URL이 HTTPS이고 **HTTP→HTTPS 301 리다이렉트**가 실제로 있는가. 자체서명 인증서가 아닌가(공식 문구가 self-signed를 명시 배제). (MUST 3)

### Step 4 — L3 크롤러·완성도 렌즈

1. **`robots.ts` 실측** — 파일 문자열이 아니라 **`/robots.txt`를 실제로 요청해** 응답 본문을 읽는다. `Mediapartners-Google` 차단·전역 `Disallow: /` 여부. (MUST 4)
2. **공개 접근** — 비밀번호·베타 게이트 없이 익명으로 열리는가. (MUST 3)
3. **내비게이션 실주행** — Playwright로 **주요 경로를 실제로 이동**한다. soft-404(200인데 404 본문)를 별도 확인한다. 공식 금지 문구는 **`"Linking to content that doesn't exist"`** 이며 이는 "위반 소지"가 아니라 **명시적 금지**다. 링크 404는 **0건**이어야 한다. (MUST 9)
   - 함께 본다: 무관·오도 페이지로의 리다이렉트, **내비게이션으로 착각되는 위치의 광고**(48182가 명시 금지).
4. **로케일 이탈** — `/en/...`에서 링크를 눌렀을 때 한국어로 튕기거나 404가 나지 않는가. (MUST 9)
5. **번역 키 노출** — 화면에 `Home.title` 같은 원문 키나 빈 문구가 보이면 미완성 화면으로 판정한다. (MUST 5·9)

### Step 5 — 수정 → 재심사 루프 (최대 3라운드)

```
R1 감사 → blocker/risk 지적 → 담당 엔지니어 수정 → R2 감사(회귀 검증 우선) → 수정 → R3 감사 → 확정
```

**라운드 규칙**

- **각 라운드는 이전 라운드 리포트의 회귀 검증부터** 시작한다. "고쳤다"는 보고를 증거로 삼지 않고 **화면에서 직접 확인**한다.
- 리포트는 **라운드별 파일**(`_workspace/06_adsense_audit_r{N}.md`)로 남긴다. 덮어쓰지 않는다 — 라운드 간 판정 변화가 감사 대상이다.
- **수정은 담당 엔지니어가 한다.** 심사관이 직접 고치면 자기가 고친 것을 자기가 승인하게 된다.
- 라운드마다 **새 지적이 나오는 것은 정상**이다(수정이 새 화면을 만든다). 다만 **3라운드에서 새 blocker가 나오면** 그건 설계 결함 신호이므로 사용자에게 에스컬레이션한다.

**종료 조건**

| 상태 | 처리 |
|---|---|
| 전 렌즈 `approve` | **통과**. 배포·완료로 진행 |
| `approve-with-risk`만 남음 | 통과하되 **잔여 리스크를 보고서에 명시**하고 사용자에게 알린다 |
| 3라운드 후에도 `blocker` 잔존 | **미통과 확정**. 잔여 항목을 "미해결(사용자 판단 필요)"로 리포트에 남기고 **에스컬레이션**. 임의 통과 금지 |

> **3라운드 상한은 무한 루프 방지용이지 "3번 하면 통과"라는 뜻이 아니다.** 상한에 도달했는데 blocker가 남았다면 그것은 통과가 아니라 **미해결**이다.

## 판정 규칙 (심사관 공통)

- **증거 없는 지적은 무효** — 정책 근거(MUST 번호 + 공식 URL) · 증거(스크린샷/DOM/`파일:라인`) · 거절 시나리오 · 심각도 · 수정 지시. 5필드를 갖추지 못한 지적은 리포트에 쓰지 않는다.
- **애매하면 통과시키지 않는다** — 심사관의 기본값은 거절이다.
- **근거 없는 기준 창작 금지** — "글자수 300자", "글 20개", "About·Contact 필수", "도메인 나이", "심사 100% 보장". **구글은 최소 분량·글 수·트래픽·도메인 나이 기준을 제시하지 않는다.** 이런 지적은 진짜 거절 사유를 가리고 팀에게 헛일을 시킨다.
- **계측기를 먼저 의심한다** — "0건 발견"은 "결함 없음"과 "검사기 고장"을 구분하지 못한다. 검출돼야 할 대조군을 일부러 하나 넣어 잡히는지 확인한 뒤 0건을 보고한다.
- **선행 QA를 신뢰하지 않는다** — `05_qa_report.md`에서 "통과"로 적힌 항목 중 **최소 2건을 직접 재검증**한다.
